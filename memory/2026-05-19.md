## 🛠️ 발견 및 완치된 지뢰급 버그 패치 (4건)

### 1. uvicorn 자동 reload 미기동 현상 (High → Fixed)
- **원인**: 백엔드가 자동 갱신(reload) 옵션 없이 켜져 있어서 Python 코드 수정 시 실시간 반영이 불가능했던 현상
- **수정**: 기존 8002 포트 uvicorn 프로세스를 완전히 강제 킬(Kill)하고, `--reload` 스위치를 장착한 상태로 백엔드 서버를 안전하게 재부팅 완료

### 2. 백엔드 `main.py` 내 fatal os.path.join 문법 파괴 버그 (Critical → Fixed)
- **파일**: `backend/app/main.py`
- **원인**: 디렉토리 존재 여부를 파악하고 생성하는 구문에서 `os.path.join(..., exist_ok=True)`라는 불가능한 런타임 오타가 내장되어 부팅 시 크래시 지뢰가 존재함
- **수정**: 디렉토리 재구축 메서드인 `os.makedirs(..., exist_ok=True)`로 문법을 정정하여 안전 기동 성공

### 3. Zustand 히스토리 동기화 시 이력 리셋 현상 (Medium → Fixed)
- **파일**: `useProjectStore.ts`
- **원인**: 타임라인 상태 덮어쓰기(`setProject`) 시 actionName이 생략되면 기존 이력 스택이 덮어씌워지며 초기화되는 버그
- **수정**: 스택이 1개 초과로 쌓여 있다면 초기화하지 않고 현재 tracks/blocks 상태만 반영하도록 보정 가드 이식

### 4. block_orchestrator.py Deprecated API Warning 제거 (Medium → Fixed)
- **파일**: `backend/app/services/block_orchestrator.py`
- **원인**: Python 3.10+에서 사용이 제한되거나 경고를 유발하는 `asyncio.get_event_loop()` 호출 코드가 남아있었음
- **수정**: 최신 비동기 안전 규격인 `asyncio.get_running_loop()`로 완전히 대체하여 경고 완전 해결

## 🌟 신규 개발 및 리팩토링 완료 기능 (6건)

### 1. 📜 Photoshop 스타일 작업 내역 패널 & 단축키 시간 여행 (메모리 FIFO 리팩토링 완료)
- **내용**: `useProjectStore`에 히스토리 스택(`historyStack`) 관리 기능을 장착.
- **UI**: 상단 Transport controls에 ↺ Undo / ↻ Redo 실물 버튼 추가 및 `Ctrl+Z`, `Ctrl+Y` 전역 단축키 바인딩.
- **패널**: 좌측 사이드바 `📜 이력` 탭을 신설하여 "블록 생성", "블록 삭제" 등 작업 목록을 시각화하고, 클릭 시 해당 과거/미래 시점으로 자유롭게 시간 여행 가능.
- **최적화 리팩토링**: `MAX_HISTORY_STACK_SIZE = 50` FIFO 순환 가드를 장착하여 메모리 누수를 완전히 차단하고, 스택 언더플로우 시 `historyIndex`를 실시간 보정하는 천재적인 수학적 인덱스 캘리브레이션 알고리즘 적용 완료.

### 2. 🌟 평생 보존용 명작 아카이브 (Named Archives) (중복 이름 & 한글 가드 리팩토링 완료)
- **내용**: 1.5초 자동 순환 백업과는 격리된, 사용자가 직접 한글/영어 이름을 명명하여 저장하는 영구 보관소.
- **API**: `POST/GET/DELETE /api/projects/{id}/archives` 엔드포인트를 구축하여 디스크에 안전하게 `.json` 영구 보관.
- **UI**: 좌측 사이드바 `🌟 명작` 탭을 통해 이름 지정 보관, 목록 노출(날짜 태그 포함), 원클릭 로딩 및 복원, 삭제 연동 완료.
- **안전 UX 리팩토링**: 프론트엔드단에서 한글 자모음(`ㄱ-ㅎ`, `ㅏ-ㅣ`) 및 영문/숫자가 최소 하나라도 포함되는지 검사하는 `hasValidArchiveNameCharacter` 사전 정규식 필터 장착. 기존 이름 중복 저장 시 `confirm` 가드를 통한 Intercepting으로 오인 덮어쓰기 전면 방지 완료.

### 3. 🔌 WebSocket 지수 백오프 자동 재연결 (네트워크 회복탄력성 리팩토링 완료)
- **내용**: 백엔드 일시 중단이나 네트워크 불안정 시 WebSocket이 영구 정지하는 병목 방어.
- **최적화 리팩토링**: `WsConnectionState` 관리 객체를 도입해 재연결 타이머와 수동 클로즈(`manuallyClosed`) 여부를 완벽 추적. 1초, 2초, 4초, 8초, 최대 10초로 누적 대기하는 **지수 백오프(Exponential Backoff)** 알고리즘 및 5회 재시도 제한 가드를 완벽 이식하여 네트워크 회복탄력성을 극상으로 진화 완료.

### 4. 🧹 백엔드 임시 캐시 정제 스케줄러 & os.path.join 버그 패치 (서버 자원 정제 리팩토링 완료)
- **내용**: stems 및 exports 디렉토리에 쌓이는 캐시 파편들의 하드 디스크 잠식 예방.
- **최적화 리팩토링**: `lifespan` 기동 시 `asyncio.create_task` 백그라운드 캐시 청소 루프를 구동하여, 24시간마다 `max(mtime, ctime)` 보수적 세이프티 비교를 거친 24시간 초과 임시 WAV/MP3/MID 캐시 오디오 파일을 `os.remove`로 비동기 안전 삭제. 서버 종료 시 `task.cancel()`과 suppress 처리 완벽 구현.

### 5. ⚡ 백엔드 블록 오케스트레이터 최적화 (DRY 페이로드 공통화 및 Deprecated API 해결 완료)
- **내용**: block_orchestrator.py 내부 비동기 이벤트 루프 획득 방식 수정 및 payload 빌딩 로직 개선.
- **최적화 리팩토링**: Python 3.10+ 표준 권장 사항에 맞춰 `get_event_loop()`를 `get_running_loop()`로 안전 교체하여 런타임 warning 경고 완전 차단. 모듈 최상단 `import uuid` 통합 관리 적용. `swap_instrument`와 `repaint_block`에 흩어져 있던 중복 페이로드 필드 조립 로직을 `_build_repaint_payload()` 단일 헬퍼 함수로 통합 및 OOP 캡슐화 완성.

### 6. 🕒 프론트엔드 Job 메타 자동 소멸 및 FIFO 제한 가드 (메모리 누수 방지 리팩토링 완료)
- **내용**: 대기열 또는 진행 중이던 AI 작업(Job) 메타가 정리되지 않고 브라우저 힙에 누적되는 현상 방어.
- **최적화 리팩토링**: `JobState`, `BlockProgress` 원본 타입을 100% 백호환 유지하면서, 스토어 모듈 스코프 영역에 `cleanupTimers`와 `jobOrder` 트래커를 설계하여 상태 오염 방지. AI 작업 완료/실패 시 30초 뒤 비동기 소멸 타이머 가동 및 수동 닫기(`dismissJob`) 시 타이머 해제 보장. 새 작업 추가 시 completed/failed Job 중 가장 오래된 것부터 우선 소멸시키는 최대 20개 FIFO 가드 구조 안착.
